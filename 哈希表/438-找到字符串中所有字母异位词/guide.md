# 438. 找到字符串中所有字母异位词

## 题目描述

给定两个字符串 `s` 和 `p`，找到 `s` 中所有是 `p` 的 **字母异位词** 的子串，返回这些子串的起始索引。不考虑答案输出的顺序。

**字母异位词** 指由相同字母重排列形成的字符串（包括相同的字符串）。

**示例 1:**
```
输入: s = "cbaebabacd", p = "abc"
输出: [0,6]
解释:
起始索引等于 0 的子串是 "cba", 它是 "abc" 的字母异位词。
起始索引等于 6 的子串是 "bac", 它是 "abc" 的字母异位词。
```

## 解题思路

### 核心思想：滑动窗口 + 哈希表
这道题本质上是寻找长度与 `p` 相同，且字符计数也与 `p` 相同的子串。

1. **定长窗口**：窗口大小固定为 `p` 的长度。
2. **哈希计数**：
   - 因为只包含小写字母，可以使用长度为 26 的数组 `vector<int>(26)` 代替 `unordered_map`，效率更高。
   - 统计 `p` 中每个字符出现的次数。
3. **滑动过程**：
   - 初始化：统计 `s` 中前 `p.len` 个字符的频率。
   - 比较：如果当前窗口的频率数组与 `p` 的频率数组相同，记录起始下标。
   - 移动：
     - 移除左边的一个字符（计数减 1）。
     - 加入右边的一个字符（计数加 1）。

### 优化细节
- 比较两个长度为 26 的数组非常快，可以认为是 O(1)。
- 也可以使用一个变量 `diff` 来维护差异数量，进一步优化到不用每次循环比较数组，但这通常是锦上添花。直接比较数组在面试中完全可以接受（26 是常数）。

## 复杂度
- **时间复杂度**：O(N)，其中 N 是 `s` 的长度。每次窗口移动操作是 O(1)（比较 26 个元素）。
- **空间复杂度**：O(1)，常数级空间存储字符计数（26个整数）。
