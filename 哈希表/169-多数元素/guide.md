# 169. 多数元素

## 🧠 解题思路引导

### 1. 什么是"多数元素"？
- 出现次数大于 `n / 2` 的元素。
- 这意味着它占据了数组的一半以上。

### 2. 直观解法
- **哈希表**：统计每个元素出现的次数，遍历哈希表找到最大的。O(n) 时间，O(n) 空间。
- **排序**：如果把数组排好序，因为多数元素超过一半，那么中间那个数（下标 `n/2`）一定是它！O(n log n) 时间，O(1) 空间。

### 3. 进阶思考：能不能 O(n) 时间且 O(1) 空间？
- 想象这是一场"诸侯争霸"。
- 多数元素是强大的诸侯，他的兵力超过了其他所有人的总和。
- 如果我们采用**同归于尽**的策略：
  - 每次拿出一个兵，和非己方的兵同归于尽。
  - 最后剩下的一定是那个最强诸侯的兵。
- 这就是著名的 **摩尔投票算法 (Boyer-Moore Voting Algorithm)**。

### 4. 摩尔投票法步骤
- 维护一个 `candidate`（候选人）和一个 `count`（票数）。
- 遍历数组：
  - 如果 `count == 0`，当前元素成为新的 `candidate`，`count = 1`。
  - 如果当前元素等于 `candidate`，`count++`。
  - 如果当前元素不等于 `candidate`，`count--`（同归于尽）。
- 最后剩下的 `candidate` 就是多数元素。
