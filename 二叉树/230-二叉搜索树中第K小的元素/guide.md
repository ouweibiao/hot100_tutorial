# 230. 二叉搜索树中第K小的元素

## 题目描述

给定一个二叉搜索树的根节点 `root` ，和一个整数 `k` ，请你设计一个算法查找其中第 `k` 个最小元素（从 1 开始计数）。

## 解题思路

### 方法：中序遍历
- **核心性质**：BST 的中序遍历是升序排列的。
- **做法**：
  - 对树进行中序遍历。
  - 计数器 `count`，每访问一个节点 `count++`。
  - 当 `count == k` 时，当前节点就是目标值。
  - 可以提前返回（剪枝）。
- **时间复杂度**：O(N) 或 O(k)。
- **空间复杂度**：O(H) (递归栈)。

## 进阶
如果 BST 频繁修改（插入/删除）并且需要频繁查第 k 小的值，怎么办？
- 可以在每个节点维护以该节点为根的 **子树结点数** (Size)。
- 类似于快速选择算法：
  - 如果 `left_size == k - 1`，当前节点就是答案。
  - 如果 `left_size >= k`，答案在左子树。
  - 如果 `left_size < k - 1`，答案在右子树，找第 `k - left_size - 1` 小。
