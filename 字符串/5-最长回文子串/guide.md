# 5. 最长回文子串

## 题目描述

给你一个字符串 `s`，找到 `s` 中最长的回文子串。

**示例 1：**
```
输入：s = "babad"
输出："bab"
解释："aba" 同样是符合题意的答案。
```

**示例 2：**
```
输入：s = "cbbd"
输出："bb"
```

## 解题思路

### 方法一：中心扩展法 (Center Expansion) —— 推荐
- **核心思想**：回文串一定是对称的。我们可以枚举所有的"回文中心"，然后向两边扩展，直到无法扩展为止。
- **中心的选择**：
  - 一个长度为 `n` 的字符串，有 `2n - 1` 个中心。
  - 为什么是 `2n - 1`？
    - 字符作为中心：有 `n` 个（例如 "aba" 的 'b'）。
    - 两个字符之间的空隙作为中心：有 `n - 1` 个（例如 "abba" 的中间）。
- **流程**：
  1. 遍历字符串 `s`，对每个位置 `i`，分别尝试：
     - 以 `s[i]` 为中心的奇数长度回文串（`expand(i, i)`）。
     - 以 `s[i]` 和 `s[i+1]` 之间为中心的偶数长度回文串（`expand(i, i+1)`）。
  2. 记录最长的长度和起始位置。
- **复杂度**：
  - 时间复杂度：O(n²)。
  - 空间复杂度：O(1)。

### 方法二：动态规划 (Dynamic Programming)
- **定义**：`dp[i][j]` 表示字符串 `s[i..j]` 是否为回文串。
- **状态转移**：
  - `s[i] == s[j]` 且 `dp[i+1][j-1]` 为真（即内部也是回文），则 `dp[i][j]` 为真。
  - 边界条件：长度为 1 或 2 的子串特殊处理。
- **复杂度**：
  - 时间复杂度：O(n²)。
  - 空间复杂度：O(n²)。
- **对比**：相比中心扩展法，DP 需要额外的 O(n²) 空间，所以在本题中**中心扩展法更优**。

### 方法三：Manacher 算法 (马拉车算法)
- 可以将时间复杂度优化到 O(n)。
- 但实现复杂，面试中通常不强制要求，除非是算法岗或专门考察。普通面试写出中心扩展法即可。

## 总结
面试首选 **中心扩展法**，代码简短且空间复杂度低。
