# 399. 除法求值

## 题目描述

给你一个变量对数组 `equations` 和一个实数值数组 `values` 作为已知条件，其中 `equations[i] = [Ai, Bi]` 和 `values[i]` 共同表示等式 `Ai / Bi = values[i]` 。每个 `Ai` 或 `Bi` 是一个表示单个变量的字符串。

另有一些以数组 `queries` 表示的问题，其中 `queries[j] = [Cj, Dj]` 表示第 `j` 个问题，请你根据已知条件找出 `Cj / Dj = ?` 的结果作为答案。

返回 **所有问题的答案** 。如果存在某个无法确定的答案，则用 `-1.0` 替代这个答案。

**示例 1：**
```
输入：equations = [["a","b"],["b","c"]], values = [2.0,3.0], queries = [["a","c"],["b","a"],["a","e"],["a","a"],["x","x"]]
输出：[6.00000,0.50000,-1.00000,1.00000,-1.00000]
```

---

## 解法：带权并查集 / 图的 DFS

### 核心思路
这道题本质上是图的连通性问题。
如果已知 `a / b = 2.0`，我们可以建立一条 `a -> b` 的边，权重为 2.0；同时建立 `b -> a` 的边，权重为 0.5。
求 `a / c`，就是寻找图中 `a` 到 `c` 的路径，并将路径上的权重相乘。

有两种主要解法：
1. **DFS/BFS**：对于每个查询，在图中搜索起点到终点的路径。适合查询较少的情况。
2. **带权并查集**：适合查询较多，或者动态添加边的场景。每个节点维护指向根节点的“比率”。

### 带权并查集原理
我们维护两个映射：
- `parent[x]`：x 的父节点。
- `weight[x]`：x / parent[x] 的值。

通过路径压缩，所有节点最终都指向同一个根节点 `root`。
此时 `weight[x]` 就变成了 `x / root` 的值。
那么 `a / b = (a / root) / (b / root) = weight[a] / weight[b]`。

### 复杂度
- 时间复杂度：并查集操作近乎 O(1)，总时间 O(N + Q)。
- 空间复杂度：O(N)。
