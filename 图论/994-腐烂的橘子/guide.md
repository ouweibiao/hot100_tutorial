# 994. 腐烂的橘子

## 题目描述

在给定的 `m x n` 网格 `grid` 中，每个单元格可以有以下三个值之一：
- 值 `0` 代表空单元格；
- 值 `1` 代表新鲜橘子；
- 值 `2` 代表腐烂的橘子。
每分钟，腐烂的橘子周围 4 个方向上相邻的新鲜橘子都会腐烂。
返回 **直到单元格中没有新鲜橘子为止所必须经过的最小分钟数**。如果不可能，返回 `-1`。

**示例 1：**
```
输入：grid = [[2,1,1],[1,1,0],[0,1,1]]
输出：4
```

---

## 解法：多源 BFS（广度优先搜索）

### 核心思路
这道题是一个典型的**层序遍历**问题。
我们可以把所有一开始就腐烂的橘子看作是**第 0 层**。
第 1 分钟腐烂的橘子是**第 1 层**，以此类推。
我们需要求的就是最后一层是多少。

由于有多个起始点（可能有多个腐烂橘子），我们需要用**多源 BFS**。
这和单源 BFS 几乎一样，唯一的区别是：**初始时将所有源点（腐烂橘子）都加入队列**，而不是只加一个。

### 算法步骤
1. 遍历网格，把所有腐烂橘子坐标加入队列，并统计新鲜橘子总数。
2. 只要队列不空且还有新鲜橘子：
   - 记录当前队列长度 `size`（当前层的节点数）。
   - 循环 `size` 次（处理完这一层）：
     - 取出队头，向四周扩散。
     - 如果遇到新鲜橘子，将其标记为腐烂，加入队列，新鲜橘子数减一。
   - 时间 +1。
3. 最后检查新鲜橘子是否为 0。

### 复杂度
- 时间复杂度：O(MN)。
- 空间复杂度：O(MN)。
